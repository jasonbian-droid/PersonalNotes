Inbound Email Processing Pipeline

1
Email Arrives → Stored in S3
AWS SES receives an inbound email at *@leads.eliseai.com and drops the raw message into S3 bucket eliseai-lead-emails. An SQS notification triggers the Lambda processor.
AWS SESS3SQSLambdaaws_tools/email_processor.py
2
First Entry Recorded → all_mail Table
email_processor.process() parses the raw email and resolves the building_id from the recipient address. A row is immediately inserted via _insert_to_all_mail() — a durable record of receipt before any further processing.
email_processor.process()_insert_to_all_mail()building_id resolutionall_mail table
3
Classifier Runs
Email body + metadata are passed to handle_classify() (in eliseai_classify), which extracts structured lead info — name, email, building, intent, etc. — as classifier_output.
⚠If classification fails, the message ID is written to not_classified_email and processing stops.
handle_classify()eliseai_classifyclassifier_outputnot_classified_email
4
Sent to Event API
send_to_event_api() constructs an InboundEmailAction and calls create_event_api_action() to POST it to the Event API service.
send_to_event_api()InboundEmailActioncreate_event_api_action()HTTP POST
5
Event API Processes into State
RequestHandler.handle() receives the action and delegates to EventApi.handle_action(), which looks up or creates the conversation state via StateIO, runs the conversation engine (AI response generation, scope resolution), and returns a state_id and multiple_conversation_id.
RequestHandler.handle()EventApi.handle_action()StateIOstate_idmultiple_conversation_id
